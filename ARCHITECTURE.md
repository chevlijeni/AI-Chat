# Project Architecture & Code Flow

This document explains how the Jeni AI Chat project is structured and how data flows through the application.

## Directory Structure

```text
/var/www/AI_chat/
├── src/                # Backend Source Code (TypeScript)
│   ├── config/         # App configurations (Database, Gemini, Passport)
│   ├── controllers/    # Request handlers (Business logic coordination)
│   ├── routes/         # API Route definitions
│   ├── services/       # Core logic (Database CRUD, AI generation)
│   ├── utils/          # Helper functions (Error handling)
│   └── server.ts       # App entry point
├── public/             # Frontend Assets
│   ├── index.html      # Main UI structure
│   ├── style.css       # Visual styling
│   └── script.js       # Client-side logic
├── dist/               # Compiled JavaScript (Auto-generated by build)
├── .env                # Environment variables (Private)
└── package.json        # Dependencies and scripts
```

## How the Code Works (Step-by-Step)

### 1. The Entry Point (`src/server.ts`)
The server starts here. It initializes the Express app, connects to the MySQL database, sets up Google OAuth sessions, and mounts all the API routes under `/api` and `/auth`.

### 2. Request Flow (Example: Sending a Chat)
When you type a message and hit "Send":

1.  **Frontend (`public/script.js`)**: Captures the input. It checks if there is a `currentSessionId`. It sends a `POST` request to `/api/chat` with the message and the session ID.
2.  **Routes (`src/routes/chatRoutes.ts`)**: Receives the request and passes it to the `ChatController`.
3.  **Controller (`src/controllers/chatController.ts`)**: 
    - Verifies the user is logged in.
    - Extracts `messages` and `sessionId` from the request.
    - Calls `ChatService.generateResponse`.
4.  **Service (`src/services/chatService.ts`)**:
    - Saves your user message to the database via `dbService`.
    - Fetches the *entire* chat history for that session from the database.
    - Sends the history to the **Google Gemini API**.
    - Once Gemini replies, it saves that AI message back to the database.
5.  **Response**: The Controller sends the AI's reply back to the frontend.
6.  **Frontend**: Updates the UI using `renderMessages()` to show the new messages.

### 3. Multi-Session Logic
- **Sessions**: Every chat you start is a "Session" in the `chat_sessions` table.
- **Filtering**: When you click a chat in the sidebar, the frontend tells the backend "Give me history for Session #5". The backend runs a SQL query: `SELECT * FROM chat_history WHERE session_id = 5`.
- **Isolation**: This ensures that when you talk to the AI in "Chat A", it doesn't remember what you said in "Chat B".

### 4. Editing a Message (Time Travel)
This is a unique feature of this app:
1. When you edit a message, the backend goes to that specific message in the database.
2. It deletes all messages that happened *after* it.
3. It updates the text of your message.
4. It tells Gemini: "Here is the conversation up to this point, now give me a new response."
5. This effectively "branches" the timeline.

## Technologies Used
- **TypeScript**: Provides type safety (catches bugs before they happen).
- **Passport.js**: Handles the complex "handshake" with Google for login.
- **MySQL2**: High-performance database driver.
- **Marked**: Converts the AI's "Markdown" text into beautiful HTML formatting.
- **SweetAlert2**: Provides the modern, styled popups for logout and deletion.
